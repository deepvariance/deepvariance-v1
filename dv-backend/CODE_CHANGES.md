# Code Changes for Directory Restructure

## Overview

Updated all code references to align with the new directory structure where:

- Test files moved to `tests/` directory
- AutoGluon models stored in `models/autogluon_models_*/`
- ML Pipeline models stored in `models/ml_pipeline_model_*.pkl`
- Unused directories removed

---

## Modified Files

### 1. services/ml_pipeline_service/ml_pipeline/llm_agents.py

**Line ~1167:** Updated AutoGluon model output path

```python
# BEFORE
model_path = f'autogluon_models_{int(time.time())}'

# AFTER
model_path = f'../../models/autogluon_models_{int(time.time())}'
```

**Reason:** AutoGluon models generated by ML Pipeline Service should be saved to backend's `models/` directory, not service root. Path is relative to `services/ml_pipeline_service/` directory.

---

### 2. services/ml_pipeline_service/app/pipeline_adapter.py

**Line ~151:** Updated ML Pipeline model save path

```python
# BEFORE
model_path = os.path.join(os.getcwd(), 'models', model_filename)

# AFTER
# Path relative to ml_pipeline_service/app/ directory
model_path = os.path.join(os.path.dirname(__file__), '../../..', 'models', model_filename)
model_path = os.path.abspath(model_path)  # Resolve to absolute path
```

**Reason:** ML Pipeline Service runs from `services/ml_pipeline_service/` directory. Need to traverse up to backend root to access `models/` directory. Using `os.path.abspath()` ensures correct path resolution.

---

### 3. .gitignore

**Updated patterns:**

```gitignore
# BEFORE
```

**Reason:** AutoGluon models are now inside `models/` directory. Removed `ml_pipeline/` pattern as that directory was deleted.

**Added patterns:**

```gitignore
# ============= Testing =============
```

**Reason:** Test files moved to `tests/` directory need proper gitignore patterns.

## Path Resolution Reference

### Directory Structure

```
dv-backend/
├── services/
│   └── ml_pipeline_service/
│       ├── app/
│       │   └── pipeline_adapter.py    (cwd when running)
│       ├── ml_pipeline/
│       │   └── llm_agents.py
│       └── start.sh
└── models/
    ├── autogluon_models_*/           (generated by ML service)
    ├── ml_pipeline_model_*.pkl       (generated by pipeline adapter)
    └── generated_model_*.py          (generated by DL training)
```

### Relative Paths

From `services/ml_pipeline_service/ml_pipeline/` (llm_agents.py):

```python
'../../models/autogluon_models_*'
```

From `services/ml_pipeline_service/app/` (pipeline_adapter.py):

```python
'../../../models/ml_pipeline_model_*.pkl'
# Or using __file__:
os.path.join(os.path.dirname(__file__), '../../..', 'models', filename)
```

## Migration Notes

### Backwards Compatibility

- **Existing models:** Old autogluon*models*\* directories were physically moved to `models/` during restructure
- **Model references:** Database records with `model_path` should be relative to backend root, so paths like `models/autogluon_models_*/` will work
- **No schema changes:** Database schema unchanged, only file system paths affected

### Future Model Paths

- All new AutoGluon models will be created in `models/autogluon_models_*/`
- All new ML Pipeline models will be saved as `models/ml_pipeline_model_*.pkl`
- All generated DL models continue using `models/generated_model_*.py`
